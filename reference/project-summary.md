# 深度强化学习智能体 Donkey Kong 项目总报告

## 一、项目背景

这是 **DTS307TC 强化学习课程作业**。任务是从零实现一个深度强化学习算法，训练智能体玩 Atari 游戏《大金刚》（Donkey Kong），撰写技术报告。

核心约束：
- **必须从零手写算法**，不能用 Stable-Baselines 等 RL 专用库（但可以用它做基准对比）
- 需要训练出"有竞争力"的表现
- 需要写一份不超过 3000 词的英文技术报告

## 二、为什么选择 Donkey Kong

Donkey Kong 是 Atari 2600 平台上**最具挑战性的游戏之一**：

- 游戏有 4 种完全不同的关卡（滚筒关、传送带关、电梯关、铆钉关）
- 敌人运动和桶的弹跳轨迹各不相同，智能体需要学习不同策略
- 奖励非常稀疏——只有跳过桶、爬梯子、收集道具才得分
- 经典 DQN 算法在这个游戏上通常只拿几百分，人类平均约 8000 分

选择这个游戏的好处：**失败有话说，成功有亮点**。即使分数不高，分析"为什么难"本身就是很好的报告素材；如果训练效果好，更能证明算法设计的有效性。

## 三、算法选择：PPO + ICM

### 3.1 什么是 PPO

PPO（Proximal Policy Optimization）是目前最主流的深度强化学习算法之一：

- **Actor-Critic 架构**：Actor（演员）决定做什么动作，Critic（评论家）评估当前局面好坏
- **随机策略**：Actor 不是直接选"最好的动作"，而是输出一个概率分布，然后随机采样。这让智能体天生具有探索能力
- **裁剪更新**：每次更新策略时，限制变化幅度（不超过 10%），防止"学歪了"

对比 DQN（2015 年的经典算法，用 ε-greedy 随机探索），PPO 的随机策略在 Donkey Kong 的复杂环境中**探索效率高得多**。

### 3.2 什么是 ICM

ICM（Intrinsic Curiosity Module，内在好奇心模块）是 PPO 的增强组件：

- **核心思想**：如果环境奖励太稀疏（比如打了几百步才得 1 分），智能体需要"好奇心"来推动它探索
- **工作原理**：ICM 包含两个小网络——
  - **前向模型**：根据当前画面和动作，预测下一帧画面。如果预测错了（画面出乎意料），说明这是"新鲜"的状态
  - **逆模型**：根据前后两帧画面，反推动作。这个网络让 ICM 学会忽略"不可控"的视觉变化
- **内在奖励**：前向模型的预测误差 = 好奇心奖励。智能体看到新东西 → 高奖励 → 驱动探索

### 3.3 为什么不用其他方案

| 方案 | 被拒绝的原因 |
|------|-------------|
| DQN + 扩展 | Donkey Kong 上表现极差（~300 分），分析"为什么失败"不如分析"怎么成功" |
| Rainbow DQN | 6 个组件组合，太复杂，容易实现错误 |
| PPO 纯（无 ICM）| 作为消融实验保留，对比 ICM 的贡献 |
| RND（另一种探索方法）| 比 ICM 更难调参 |

## 四、智能体的"大脑"架构

```
游戏画面（4 帧 × 84×84 像素）
        ↓
  ┌─────────────┐
  │  CNN 编码器  │  三层卷积网络，提取视觉特征
  │  输出: 256维 │
  └──────┬──────┘
         │
    ┌────┼────┐
    ↓    ↓    ↓
  Actor Critic ICM
  │     │     │
  │     │     ├── 逆模型：根据前后两帧，反推动作
  │     │     └── 前向模型：预测下一帧画面
  │     │           预测误差 → 好奇心奖励
  │     └── 输出 1 个数：这个局面的"价值"
  └── 输出 18 个数：18个按键的概率分布
```

- **CNN 编码器**：就像眼睛，把像素转换成"有意义的信息"
- **Actor**：根据当前画面，决定按哪个键（18 个键对应 Atari 手柄全部按钮）
- **Critic**：估计"现在这局面好不好"，用来评判 Actor 的选择
- **ICM**：给 Actor 额外的"好奇心奖励"，驱动探索新区域

## 五、训练过程

### 5.1 训练循环

```
第 1 步：8 个游戏实例同时运行，各玩 128 步 → 收集 1024 条经验
第 2 步：ICM 计算每条经验的好奇心奖励
第 3 步：GAE 算法计算每条经验的"真正价值"（考虑未来收益）
第 4 步：PPO 更新策略网络（重复 4 轮）
第 5 步：ICM 更新好奇心网络（分批处理，防显存溢出）
第 6 步：每 500 局保存一次模型（断点续训用）
第 7 步：回到第 1 步，直到跑完 5000 万步
```

### 5.2 训练中的关键数据

训练过程中观察这几个指标：

| 指标 | 含义 | 健康范围 |
|------|------|----------|
| **reward** | 原始游戏分数（最近 100 局均值） | 持续上升 |
| **entropy** | 策略熵，衡量"探索程度" | 2.5 → 1.0 缓慢下降 |
| **policy_loss** | Actor 的学习损失 | -0.01 ~ -0.04 |
| **icm_loss** | 好奇心网络的学习损失 | 1.0 → 逐渐上升（正常！） |
| **fps** | 每秒处理多少帧 | 越高越好 |

### 5.3 Donkey Kong 分数等级

| 分数 | 水平 |
|------|------|
| 0-100 | 没学到任何东西 |
| 100-500 | 偶尔跳过桶 |
| **500-2000** | **及格——展现清晰的强化学习效果** |
| 2000-5000 | 良好——多阶段学习 |
| 5000+ | 优秀——超过大部分文献 |

### 6.2 训练中遇到的 Bug 及修复

1. **ALE 环境注册失败** → 需要在 gymnasium 前先导入 ale_py + shimmy
2. **双重帧跳冲突** → 新版 gymnasium 需要传 frameskip=1
3. **PPO 更新时计算图崩溃** → 未 detach 的 advantage 导致重复反向传播
4. **checkpoints/ 目录不存在** → 加了自动创建目录的代码
5. **ICM 更新导致显存溢出** → 根本原因：ICM 把全部 1024 个样本一次性塞进 CNN，虽然是 no_grad 但中间激活层占用大量显存。修复：ICM 改成每 256 个样本一批处理
6. **SSH 断连导致终端乱码** → tmux 保护了训练进程不中断
