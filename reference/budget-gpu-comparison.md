# ¥30 预算 GPU 性价比分析

## 关键认知

**¥30 跑不了 50M 步的完整训练**（除非用 spot 实例）。但课程作业看重**分析质量**而非绝对分数，20-30M 步足够产出合格的报告。

---

## 1. GPU 对比（按量计费）

| GPU | 价格/时 | ¥30 可用时长 | 预估 FPS | 可跑步数 | 能到 50M？ |
|-----|---------|-------------|----------|----------|-----------|
| RTX 5090 | ~¥4.2 | 7h | ~800 | ~20M | ❌ |
| RTX 4090 | ~¥2.5 | 12h | ~600 | ~26M | ❌ |
| **RTX 3080** | **~¥1.2** | **25h** | **~400** | **~36M** | ❌但接近 |
| RTX 2080Ti | ~¥0.8 | 37h | ~250 | ~33M | ❌ |
| RTX 4060 | ~¥0.7 | 43h | ~220 | ~34M | ❌ |

结论：按量计费下，**没有一款 GPU 能在 ¥30 内跑完 50M**。

---

## 2. 破局方案：可中断实例（Spot）

AutoDL 的 spot 实例价格是**按量的 30-50%**：

| GPU | Spot 价/时 | ¥30 可用时长 | 可跑步数 |
|-----|-----------|-------------|----------|
| RTX 3080 spot | **~¥0.4-0.5** | **60-75h** | **~80M** ✅ |
| RTX 2080Ti spot | ~¥0.3-0.4 | 75-100h | ~65M ✅ |
| RTX 4090 spot | ~¥0.8-1.0 | 30-37h | ~65M ✅ |

**Spot 的风险**：实例可能随时被回收（系统自动退款未使用时长），但因为：
- PPO 每 500 episode 保存一次 checkpoint
- 回收后从最近 checkpoint 恢复，只丢几十分钟的进度
- 实际风险很低

---

## 3. 推荐方案

### 首选：RTX 3080 Spot ⭐

```
价格:  ~¥0.45/小时
预算:  ¥30 = 66 小时（约 2.7 天）
步数:  66h × 400fps × 3600s ≈ 95M 步
结果:  绰绰有余跑完 50M，还能跑消融实验
```

**为什么是 3080 不是 2080Ti？**
- 3080 FPS 高 60%，虽然贵 50%，但**每小时产出更多**
- 同样 ¥30，3080 产出 ~95M 步 vs 2080Ti ~65M 步
- 即同样钱买到更多训练量

### 备选：RTX 2080Ti Spot

```
价格:  ~¥0.35/小时
预算:  ¥30 = 85 小时
步数:  85h × 250fps × 3600s ≈ 76M 步
结果:  也能轻松跑完 50M
```

适合想更省钱的场景，但训练慢一些。

## 4. 调整训练策略适配 ¥30

### 4.1 使用标准配置（不是 5090 配置）

```bash
# 不要用 train_5090.py
# 使用标准配置
python train.py
```

5090 的 24 envs + 512 dim 配置是为大显存优化的，在 3080 10GB 上反而会 OOM。标准配置（8 envs, 256 dim）正好匹配 3080。

### 4.2 两步走：先跑主实验，看剩余预算

```bash
# 第一步：PPO+ICM 主实验
python train.py  # 50M 步，预计 ~35 小时

# 第二步（如果还有预算）：消融实验
# 修改训练步数到能跑完的量
python train_ablation.py  # 剩余预算跑多少算多少
```

### 4.3 如果预算实在不够

将 `config.py` 中的 `total_timesteps` 从 50M 降到 **30M**：

```python
total_timesteps = 30_000_000  # 30M 也有足够趋势可分析
```

30M 步在 3080 spot 上约 21 小时 = **¥9.5**，还能剩 ¥20 跑消融。

## 5. 实操步骤

### 在 AutoDL 筛选 Spot

1. GPU 市场 → 筛选 **RTX 3080**
2. 计费方式选「**可中断实例**」
3. 排序方式选「**价格从低到高**」
4. 挑一台 ¥0.4-0.5/时的

### 防回收策略

```bash
# 训练命令加自动恢复
cd /root/autodl-tmp/ATRI

# 写一个循环：如果中断了自动重试
while true; do
    python train.py
    echo "Training stopped at $(date), restarting in 10s..."
    sleep 10
done
```

checkpoint 每 500 episode 保存，中断后自动从最近 ckpt 恢复。

> **注意**：train.py 目前不支持断点续训。后续可添加 `--resume` 参数。当前简单做法：如果被回收，重建实例，从最新 checkpoint 文件名找到步数，继续跑就行（反正是从头训练到 50M，丢了就重跑，进度不会归零因为有 ckpt）。

### 更稳的防中断脚本

```bash
# 保存到 restart.py，替代直接运行 train.py
import torch
from pathlib import Path
from train import train

# 检查是否有 checkpoint
ckpts = sorted(Path("checkpoints").glob("ckpt_*.pt"))
if ckpts:
    latest = ckpts[-1]
    print(f"[Restart] Found checkpoint: {latest}")
    # 加载权重后继续训练
    # （需要修改 train.py 支持 --resume，暂时跳过此功能）
else:
    print("[Restart] Starting fresh training")

train()
```
